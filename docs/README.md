## 🚀 v3.4.3

本次更新带来了令人兴奋的改进，特别是针对阿里云OSS存储的上传功能。我们引入了**暂停和继续**上传的能力，这不仅极大地提升了用户体验，还增强了代码的可维护性。想象一下，在处理大文件或网络不稳定时，能够随时暂停上传并在条件允许时继续，这将是一个多么便捷的功能！

### 变更摘要

本次版本更新主要集中在以下几个方面：

-   **🆕 新功能**：支持阿里云OSS存储上传的暂停和继续功能，让用户能够更灵活地控制上传过程。
-   **✨ 功能优化**：优化了上传进度监控和状态管理，确保用户能够实时了解上传状态，并在必要时进行操作。
-   **🐞 Bug修复**：修复了上传过程中可能出现的异常处理问题，提升了应用的稳定性。

### 详细变更

#### 🆕 新功能：支持阿里云OSS存储上传的暂停和继续

在这次更新中，我们引入了**暂停和继续上传**的功能，这一改进极大地提升了用户体验和代码的可维护性。具体来说：

-   **暂停上传**：当用户需要暂停上传时，系统会记录当前的进度点（checkpoint），并允许用户在稍后继续上传。
-   **继续上传**：用户可以在网络条件改善或其他合适的时候，继续之前暂停的上传任务，无需重新开始。

这一功能特别适用于处理大文件或在网络不稳定的环境中上传文件。用户不再需要担心因网络问题导致上传失败或需要重新开始，从而节省了时间和流量。

#### ✨ 功能优化：优化上传进度监控和状态管理

为了更好地支持暂停和继续功能，我们对上传进度监控和状态管理进行了优化：

-   **实时进度更新**：系统会实时更新上传进度，并通过`upload-progress`事件将进度信息传递给前端，确保用户能够随时了解上传状态。
-   **状态管理**：我们引入了`UploadsContext`来管理上传状态，包括上传进度、状态（如`uploading`、`paused`、`completed`等）以及错误信息。

这些优化不仅提升了用户体验，还使得代码更加清晰和易于维护。

#### 🐞 Bug修复：修复上传过程中可能出现的异常处理问题

在本次更新中，我们修复了上传过程中可能出现的异常处理问题：

-   **异常处理**：当上传过程中出现异常（如`AbortError`或`CancelError`）时，系统会正确处理这些异常，并更新上传状态为`paused`，同时向用户显示相应的提示信息。
-   **状态恢复**：在异常处理后，用户可以选择继续上传，系统会根据之前的进度点（checkpoint）恢复上传任务。

这些修复确保了上传过程的稳定性和可靠性，减少了用户在上传过程中遇到的问题。

### 总结

本次版本更新带来了许多重要的改进，特别是支持阿里云OSS存储上传的暂停和继续功能。这一功能不仅提升了用户体验，还增强了代码的可维护性。我们相信，这些改进将帮助用户更高效、更方便地管理他们的在线云存储。

感谢所有用户的支持和反馈，我们将继续努力提升产品的质量和用户体验！
